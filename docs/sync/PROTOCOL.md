# פרוטוקול שיתוף פעולה — Claude × Codex × OR

> **מקור-אמת יחיד** לכל כללי העבודה המשותפת. שני הסוכנים קוראים את הקובץ הזה דרך ה-import
> ב-`CLAUDE.md` (Claude) וב-`AGENTS.md` (Codex). אל תשכפל כללים במקום אחר — שנה כאן.

---

## 0. הזהויות

שלושה שחקנים:
- **`claude`** — Claude Code (Anthropic, Opus 4.8). הסוכן המתכנן / מבקר / פורס (deploy).
- **`codex`** — OpenAI Codex CLI (GPT-5.5). הסוכן המממש המהיר.
- **`OR`** — האדם. המתזמר (orchestrator) ובעל ההחלטות.

**OR מחזיק את ה"טוקן" בברירת מחדל ומחלק אותו.** סוכן עובד על הקוד **רק כשהטוקן שלו**.

## 2. ריטואל החתימה — START / DURING / END

### START (תחילת עבודה, כל תור)
1. ודא שהטוקן שלך — שורת "סבב נוכחי" ב-`BOARD.md`. אם לא תורך → עצור, חכה ל-OR.
2. קרא את כל `BOARD.md` ואת ~15 השורות האחרונות של `CHANNEL.md`.
3. סשן חדש? Claude — ה-hook הזריק `STATE.md` + `BOARD.md` אוטומטית. Codex — קרא ידנית `docs/handoff/STATE.md` ואת `docs/sync/`.
4. הצהר משימה: עדכן את שורת ה-`Now` שלך ב-`BOARD.md` עם **הקבצים המדויקים** שתיגע בהם.
5. בדוק התנגשות: קובץ שאתה צריך מופיע ב-`Now` של הסוכן השני? **אל תיגע** — פוסט ב-`CHANNEL.md` ובקש.
6. פוסט פתיחה ב-`CHANNEL.md`: `[ts] <author> → <other>: תפסתי <קבצים>. מתחיל <משימה>.`

### DURING (תוך כדי עבודה)
7. ערוך **רק** קבצים שמופיעים תחת ה-`Now` שלך. צריך עוד קובץ? עדכן `Now` קודם, ואז גע בו.
8. החלטת ארכיטקטורה / שאלה חוסמת / לפני deploy → פוסט `→ OR` ב-`CHANNEL.md` **ועצור**.
9. אל תכתוב ל-`docs/handoff/` ידנית.

### END (סיום יחידת עבודה)
10. **אמת**: הרץ `npm run build` (ו/או `tsc --noEmit` / בדיקות). חובה לפני שמסמנים "Done".
11. **Commit** עם קידומת מחבר: `[claude] <type>: <תיאור>` או `[codex] <type>: <תיאור>`.
12. הזז את המשימה ב-`BOARD.md` מ-`Now` ל-`Done` (עם hash + פקודת האימות שעברה). נקה את ה-`Now` שלך ל-`—`.
13. פוסט סיום ב-`CHANNEL.md`: `[ts] <author> → <other>: סיימתי, commit <hash>, build ירוק. הטוקן שלך.`
14. גבול שלב? הפעל סקיל `handoff`, ואז (Claude בלבד) `git push origin main`.

## 3. כללי בעלות על קבצים

- ערוך **רק** קבצים שתחת ה-`Now` שלך ב-`BOARD.md`.
- קובץ שמופיע ב-`Now` של הסוכן השני = **אסור לגעת**. פתור דרך `CHANNEL.md`.
- תפיסה חייבת להיות **קבצים ספציפיים או תיקייה צרה** — אף פעם לא "כל הריפו".

---

## 4. מצב סנכרון — ברירת מחדל: SEQUENTIAL (רצוף)

- **סוכן אחד בכל פעם**, OR מעביר את הטוקן, הכול על `main`. אפס התנגשויות merge.
- **מעבר זמני למקביל** רק כשכל אלה מתקיימים: (א) שתי משימות עצמאיות לגמרי בשכבות נפרדות
  (`netlify/functions/**` מול `src/**`), בלי שום קובץ משותף; (ב) OR מאשר במפורש ויבורר את ה-merge;
  (ג) העבודה גדולה מספיק שעבודה רצופה מבזבזת סשן שלם.
  במקביל: branch-per-agent → PR → הסוכן השני מבקר → OR ממזג.

## 5. קומיטים

- קידומת מחבר חובה: `[claude]` / `[codex]`.
- פורמט: `[claude] feat: <תיאור בעברית>`. `<type>` ∈ `feat | fix | perf | chore | docs`.
- (תואם להיסטוריה הקיימת: `feat: Phase 3…`, `fix: ESLint…`.)
- בדיקת עבודת הסוכן השני: `git log --grep='\[codex\]'`.
- git כבר מוגדר נכון בתחנה הזו (`safe.directory` + זהות). בשיבוט חדש:
  `git config --global --add safe.directory "C:/Users/OR/קודקס/משאוושה/Meshausha"`.

## 7. חלוקת עבודה — Relay (ברירת מחדל)

| | **claude** | **codex** |
|---|---|---|
| תפקיד | מתכנן + מבקר + deploy | מממש מהיר לפי התוכנית |
| חוזק | review ב-xhigh, superpowers (תכנון/TDD), MCP של GitHub + Netlify, Playwright | הקשר ענק (372k), node_repl (הרצת JS), איטרציה מהירה |

זרימה: claude מתכנן ומכתיב משימה → codex מממש → claude מבקר diff → claude עושה deploy.
**רק claude עושה deploy** (בעלות יחידה מונעת double-deploy).
