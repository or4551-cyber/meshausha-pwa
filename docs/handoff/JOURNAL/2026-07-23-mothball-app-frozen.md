## הדרך: מה נוסה, מה עבד, מה נכשל ולמה
1. **חקירה ראשונית הפריכה הנחה מרכזית:** חיפוש בכל הריפו — אין Supabase בכלל. כל הדאטה ב-Netlify Blobs על אותו אתר. ההוצאה החודשית היחידה: Netlify Pro (~$19), שמכסה את **כל 25 האתרים בחשבון** (duet, 2d-light, tzavella...), לא רק משאוושה.
2. **אי-הבנה שתוקנה ע"י OR:** הצגתי שאלת "לרדת מ-Pro או להישאר" — OR הבהיר שהכוונה לא לכסף אלא לחסימת גישה. המנוי לא שונה.
3. **גיבוי:** `netlify env:list --json` (יצא עם BOM של PowerShell → שבר `JSON.parse` ב-Node; תוקן ב-`.replace(/^\uFEFF/,'')`). דמפ ה-blobs נעשה בסקריפט Node עם `@netlify/blobs` (מהיר ממאות קריאות CLI); `API_TOKEN` יצא ריק מה-env של Netlify — הערך קיים ב-`.env` המקומי שהועתק לארכיון.
4. **המלכודת הגדולה של השלב — תזמון כפול:** אחרי כיבוי הקרונים ב-`netlify.toml` ופריסה, `netlify functions:list` עדיין הציג את הפונקציות. בדיקת grep גילתה שהתזמון מוגדר **גם** ב-`export const config` בתוך הקוד — כיבוי ה-toml לבדו לא היה עוצר כלום. הוער גם בקוד, נפרס שוב, ואומת: `schedule=` ריק בשתי הפונקציות.
5. **נעילת סיסמה דרך API:** `netlify api updateSite` עם `body.password`. שתי תקלות טכניות: PowerShell 5.1 בלע את המרכאות ב-JSON (תוקן ב-`\"` escaping), וה-API מחזיר `password: ''` גם אחרי הצלחה (מסוך). לכן האימות נעשה כמו משתמש אמיתי: GET ללא סיסמה → **401**; POST של טופס הסיסמה + GET באותו session → **200**. ✔
6. **תגלית:** שלושת האתרים הישנים (meshausha, meshausha01, meshios) עדיין שירתו גרסאות ישנות של "מערכת הזמנות משאוושה" לכל דורש. ננעלו באותה סיסמה ואומתו 401.
